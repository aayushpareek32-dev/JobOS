### 题目 4：LLM 训练全流程理解深度

**难度**：困难  
**考查重点**：LLM 原理理解、训练流程掌握、细节把控

你实现了从 Pretrain 到 DPO 的完整训练流程。请解释：

1. **SFT（Supervised Fine-Tuning）** 和 **RLHF** 的本质区别是什么？为什么有了 SFT 还需要 RLHF？
2. **DPO（Direct Preference Optimization）** 为什么能绕过 Reward Model 直接优化？它的 loss function 是什么？

> **追问方向 A（理论深度）**：在 RLHF 中，Reward Model 是如何训练的？为什么 PPO 算法在 LLM 对齐场景下需要进行特殊的裁剪（clip）？
>
> **追问方向 B（实践经验）**：在你的 Learn MiniMind 项目中，SFT 和 DPO 的训练时间、显存占用、效果差异分别是怎样的？

**参考回答要点**：
- SFT vs RLHF：SFT 教会模型"如何回答"，RLHF 教会模型"如何偏好"
- DPO Loss: $L = -\log \sigma \left( \log \frac{\pi_\theta(y^+|x)}{\pi_{ref}(y^+|x)} - \log \frac{\pi_\theta(y^-|x)}{\pi_{ref}(y^-|x)} \right)$
- PPO Clip：防止策略更新过大导致模型崩溃
- 训练成本：SFT < DPO < RLHF（PPO）

### 题目 11：Python 高级特性与工程实践

**难度**：简单  
**考查重点**：Python 基础、工程代码质量、问题排查

以下代码在运行时会遇到什么问题？如何修复？

```python
def process_items(items):
    results = []
    for item in items:
        result = item.get('data', {}).get('value', 0)
        results.append(result)
    return results

# 调用
process_items([{'data': {'value': 1}}, None, {'other': 'field'}])
```

> **追问方向 A（Python 功底）**：你了解 `dict.get()` 和 `dict['key']` 的区别吗？在什么场景下应该用哪个？
>
> **追问方向 B（代码优化）**：如果 items 列表很大（百万级），上述代码有什么性能问题？

**参考回答要点**：
- 问题：`None.get()` 会报 AttributeError
- 修复：`isinstance(item, dict)` 检查或 `item.get('data', {}) if item else 0`
- `.get()` vs `[]`：前者安全返回 None，后者抛异常
- 性能优化：列表推导式、并行处理、生成器
